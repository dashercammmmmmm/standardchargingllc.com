What the Website Does

The standardchargingllc.com website is a one-page marketing site for Standard Charging LLC — Cam's EV charging business based in Albany, OR. It has one job: convince local business owners to let Standard Charging install EV chargers in their parking lots.

Who it's talking to

The audience is local business owners — restaurants, retail shops, gyms, anyone with a parking lot. These are not tech people. They need to feel safe, see the value quickly, and know how to get in touch.

What it's trying to accomplish

  • ✅ Build credibility fast (looks professional, clean, modern)
  • ✅ Answer the biggest objection upfront: "$0 cost to you"
  • ✅ Show the process is simple and low-effort for the business owner
  • ✅ Get them to call or reach out — there's no form, just a phone number
💡 The Core Message

"We put EV chargers in your parking lot for free, drivers come to charge, they spend money in your business, and you get a cut of the charging revenue. You do nothing. We handle everything."

How to Make Common Changes

The entire website lives in one file: index.html. Open it in any text editor (VS Code, TextEdit, Notepad — anything works). All changes below are made in that file.

🛠 Quick tip before you edit

Use your editor's Find & Replace (usually Cmd+F on Mac or Ctrl+F on Windows). Search for the exact text you want to change — it's faster than scrolling through code.

📞 Change the Phone Number

The phone number appears in two places. Search for 3522867134 and you'll find both.

✏️ How to edit
  1. Open index.html in your text editor
  2. Press Cmd+F (Mac) or Ctrl+F (Windows) and search for 3522867134
  3. You'll find two hits — one in the href="tel:..." attribute, one as the display text
  4. Update both to your new number (no spaces or dashes in the href version)

Here's what the code looks like (before and after example):


<a href="tel:3522867134">📞 (352) 286-7134</a>


<a href="tel:5035551234">📞 (503) 555-1234</a>

✍️ Update the Main Headline

✏️ How to edit
  1. Open index.html
  2. Search for Turn Your Parking Lot Into a
  3. You'll see the <h1> tag with the headline
  4. Change the text — but keep the <span> tags if you want part of it in green

<h1>Turn Your Parking Lot Into a <span>Customer Magnet</span></h1>


<h1>Turn Your Parking Lot Into a <span>Revenue Machine</span></h1>
💡 The green part

Whatever text is inside <span>...</span> inside the headline will appear in green. If you want the whole headline in white, just delete the <span> tags.

➕ Add a New Benefit Card

The "Why Sponsor" section has 6 cards in a grid. You can add a 7th (or more) by copying the pattern below.

✏️ How to edit
  1. Open index.html
  2. Search for benefits-grid
  3. You'll see the <div class="benefits-grid"> container
  4. Scroll to the last </div> before the closing </div> of benefits-grid
  5. Paste a new card block right before that closing tag

Here's the exact code for a new card — copy, paste, and edit the emoji, title, and description:

<div class="benefit">
  <span class="benefit-icon">🚗</span>
  <h3>Your Title Here</h3>
  <p>Your description text here. Keep it to 1–2 sentences.</p>
</div>
💡 Emoji picker

On Mac: press Cmd + Ctrl + Space to open the emoji picker anywhere. On Windows: press Win + . (Windows key + period).

🎨 Change the Colors

All colors are defined in one place at the top of the file inside a :root block. Change a value there and it updates everywhere on the site automatically.

✏️ How to edit
  1. Open index.html
  2. Search for :root
  3. You'll see the color variables listed
  4. Change the hex value (the #xxxxxx code) for any color you want to update
/* Color variables — change these to retheme the whole site */
:root {
  --green:  #1db954;  /* Main accent: buttons, highlights, numbers */
  --dark:   #0d1117;  /* Darkest background (nav, footer, stats) */
  --mid:    #1a1a2e;  /* Hero & CTA section background */
  --light:  #f5f7fa;  /* Light gray (How It Works background) */
  --muted:  #6b7280;  /* Gray body text / descriptions */
}
💡 Finding color codes

Google "color picker" to get a hex code for any color you want. Just paste the new #xxxxxx value in place of the old one. For example, to change the green to blue: --green: #0066ff;

Where Files Live & How GitHub Pages Works

Where the files are on your machine

The website folder lives here on the Mac mini:

~/.openclaw/workspace/standard_charging/website/

Inside that folder, you'll find:

  • index.html — the entire website (the only file you need to edit)
  • .well-known/appspecific/com.tesla.3p.public-key.pem — Tesla key file (see Section 5)

What GitHub Pages is (in plain English)

GitHub Pages is a free service from GitHub that turns a folder of files into a real live website. Here's how it works, step by step:

  1. You have a repository (a folder tracked by Git) that lives both on your machine and on GitHub's servers.
  2. When you push changes to GitHub (upload your latest version), GitHub Pages automatically detects the update.
  3. Within about 1–2 minutes, GitHub Pages rebuilds the live website at standardchargingllc.com using your updated files.
  4. Anyone who visits the site now sees your new version. No hosting fees, no servers to manage.
💡 Think of it like Google Docs

You edit a file on your machine, then "sync" it (push to GitHub), and the live website updates automatically. The difference from Google Docs is you have to manually push — it doesn't auto-sync.

⚠️ Changes don't go live until you push

Editing index.html on your machine does not update the website. You have to run the git push command (see Section 6). Until you do, the live site stays as-is.

The Tesla Key File — Don't Delete It

Inside the website folder, there's a hidden-ish file at this path:

.well-known/appspecific/com.tesla.3p.public-key.pem

What it is

This is a verification file required by Tesla's network. When Tesla's app or charging network wants to confirm that standardchargingllc.com is a legitimate, trusted site (not a fake), it checks for this file at that exact URL.

Think of it like a business license posted in your window. It's not something customers notice, but the health inspector (Tesla's system) needs to see it there.

What happens if you delete it

Tesla's systems won't be able to verify the site. This could break future integration with Tesla's charging network or app visibility — which is core to the business model.

🚨 Don't delete this file

Leave .well-known/appspecific/com.tesla.3p.public-key.pem exactly where it is. Don't rename it, move it, or modify it. It needs to be accessible at that exact path for Tesla's verification to work.

Does it affect the website visually?

No. Visitors to the website never see it or interact with it. It's completely invisible to humans — only Tesla's automated systems read it.

How to Deploy Updates

After you've made changes to index.html, you need to push those changes to GitHub so the live website updates. Here's the full process — three commands total.

💡 You only need to do this after you're done editing

You can make 10 edits to the file and then run these commands once at the end. You don't need to push after every single change.

Step 1 — Open Terminal and navigate to the website folder

cd ~/.openclaw/workspace/standard_charging/website

Step 2 — Stage your changes

This tells Git "I want to include all my changes in the next save."

git add .

The . means "everything in this folder." You can also specify a single file like git add index.html.

Step 3 — Commit (save a snapshot)

This creates a saved checkpoint with a message describing what you changed. Write something short and descriptive.

git commit -m "Updated phone number and hero headline"

Step 4 — Push to GitHub (go live)

This uploads your snapshot to GitHub, which triggers the website to update within 1–2 minutes.

git push

The full sequence, copy-paste ready:

✏️ Run these three commands in order
git add .
git commit -m "Your change description here"
git push

How do I know it worked?

After running git push, wait about 60–90 seconds then refresh standardchargingllc.com in your browser. Your changes should be live.

You can also check the Actions tab on your GitHub repository page to watch the deployment progress in real time.

⚠️ If git push asks for a password

GitHub no longer accepts passwords — it uses a Personal Access Token (PAT). If you get a permission error, ask Ben to walk you through setting up a token or SSH key. This only needs to be done once.
